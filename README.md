## ☁️ 커밋 컨벤션

다음 컨벤션을 따릅니다.

| 커밋 타입 | 설명 |
|-----------|------|
| `feat`    | 새로운 기능을 추가할 경우 |
| `fix`     | 버그를 고친 경우 |
| `style`   | 코드 포맷 변경, 세미 콜론 누락, 코드 수정이 없는 경우 |
| `refactor`| 리팩토링 하는 경우 |
| `comment` | 필요한 주석 추가 및 변경 |
| `chore`   | `package.json` 변경이나 모듈을 변경했을 때 |
| `docs`    | 문서를 수정한 경우 |
| `test`    | 테스트 추가 |
| `rename`  | 파일 혹은 폴더명을 수정하거나 옮기는 작업만을 하는 경우 |
| `remove`  | 파일을 삭제하는 작업만 하는 경우 |
